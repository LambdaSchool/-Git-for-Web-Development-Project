    1. What is Semantic HTML? 
        Semantic HTML give meaning to each tag and element a HTML file so different developers can more easily understand others' code.

    2. What is HTML used for? 
        HTML is used for marking up, or organizing text content for a website, so that the browser knows how to display the text on screen.

    3. What is an attribute and where do we put it? 
        Each HTML element have many properties called "attributes", we put it inside the opening tag with attribute="value" pair convention.

    4. What is the h1 tag used for? How many times should I use it on a page?
        h1 tag is used for the title for the page, and so ideally one should only use it once on a given page.

    5. Name two tags that have required attributes
        <a href="#"></a>,  <img src="#">

    6. What do we put in the head of our HTML document? 
        Inside the <head> of our HTML document, we include information related to the document, like referencing files outside of the HTML document, CSS, Javascript files, or embedded CSS, Javascript file.
        The <title> of the page is also defined in here. Anything that is back end.

    7. What is an id? 
        An id is an attribute that makes each HTML element unique in a page.

    8. What elements can I add an id to? 
        You can give an id to any HTML elment.

    9. How many times can I use the same id on a page? 
        You can only use it once, because id must be unique.

    10. What is a class? 
        A class groups elements, can be different types, together.

    11. What elements can I add a class to? 
        You can add class to any HTML element.

    12. How many times can I use the same class on a page? 
        You can use it as many times as you want.

    13. How do I get my link to open in a new tab?
        <a href="#" target="_blank">Text</a>

    14. What is the alt attribute used for? 
        It is used to display a description of an image if the image doesn't load correctly.

    15. How do I reference an id?
        #id_name

    16. What is the difference between a section and a div
        A section has semantic meaning, telling others that things in a section belong together. A div is just a container for different elements and does not entail information to others.

    17. What is CSS used for? 
        CSS is Cascading Style Sheet, and it is used for styling HTML to be displayed on screen. It makes the website colorful and its content organized for viewability.

    18. How to we select an element? Example - every h2 on the page
        h2 {
            styling...
        }

    19. What is the difference between a class and an id? - Give me an example of when I might use each one
        A class can contain different elements, which you want to style them the same way. 
        Example, if you want to make elements in a section, say h2, and p to have same font and color, you group them in the same class and select the class to style them together.

        An id can only be used on one element, hence it has a higher specificity weight than class. You can style one specific element, but it is generally used for referencing to that specific tag for Javascript, or move your window to that tag.

    20. How do we select classes in CSS?
        .class_name {
            styling...
        }

    21. How do we select a p element with a single class of “human””?
        p.hunman {
            styling...
        }

    22. What is a parent child selector? When would this be useful? 
        The parent refers to the outer tag, and the child refers to the nested tag inside the parent. This inheritence gives CSS its name as styling the parents automatically style their children.
        The child selector allows the children to have specific attributes from their parents.

    23. How do you select all links within a div with the class of sidebar?
        div.sidebar a{
            styling...
        }

    24. What is a pseudo selector?
        The pseudo selector is a keyword in CSS that indicates the state of the element.

    25. What do we use the change the spacing between lines?
        line-height: #;

    26. What do we use to change the spacing between letters?
        letter-spacing: normal;

    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        To capilize everything: text-transform: uppercase;
        To lower case everything: text-transform: lowercase;
        To titlecase everything: text-transform: capitalize;

    28. How do I add a 1px border around my div that is dotted and black?
        div {
            border-style: dotted;
            border-width: 1px;
            border-color: black;
        }

        or shorthand

        div {
            border: 1px dotted black;  //width, style, color
        }

    29. How do I select everything on the page? 
        * {
            styling...
        }

    30. How do I write a comment in CSS?
        // for single line, and /* Comment here */ for multiple lines.

    31. How do I find out what file I am in, when I am using the command line? 
        pwd for print working directory

    32. Using the command line - how do I see a list of files/folders in my current folder?
        ls for list, can use modifier -l for details, and -a for showing hidden files

    33. How do I remove a file via the command line? Why do I have to be careful with this? 
        rm file_name
        You have to be careful because once you execute the command, the file is gone forever.

    34. Why should I use version control? 
        To keep track of different saved versions of your project, allow people in a project to work on their own branch, and merge to the master branch when they finish.
        It also makes it easy to keep track of what content were added, edited or deleted.

    35. How often should I commit to github?
        You should commit as often as you can when you are working locally. However, you should push to github when you think you are at the point for others to review your work.
        Then you can submit a pull request to merge your branch to the master branch, for example.

    36. What is the command we would use to push our repo up to github?
        git push -u origin your_branch
